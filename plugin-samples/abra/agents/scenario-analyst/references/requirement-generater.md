# AI 에이전트 요구사항 생성기

> 사용자가 **서비스 목적**과 **생성 갯수**만 입력하면,
> Claude Code가 다양한 관점의 요구사항을 자동 생성함.

---

## 사용법

아래 프롬프트의 `{{서비스_목적}}`,  `{{생성_갯수}}`, `{{결과파일 디렉토리}}`를 교체하여 Claude Code에 입력.

---

## 프롬프트

```
당신은 비즈니스 요구사항 전문가입니다.
주어진 서비스 목적을 분석하여, AI 에이전트 구축을 위한
요구사항을 서로 다른 관점으로 {{생성_갯수}}개 작성해주세요.
결과는 {{결과파일 디렉토리}}에 마크다운으로 작성하세요.  
 
# 서비스 목적
{{서비스_목적}}

# 생성 규칙

## 관점 다양화
각 버전은 아래 관점 중 서로 다른 조합으로 작성:
- 업무 자동화 중심: 반복 업무 제거, 처리 속도 향상에 초점
- 고객 경험 중심: 사용자 만족도, 응답 품질, 접근성에 초점
- 비용 절감 중심: 인력 대체, 운영 비용 감소에 초점
- 의사결정 지원 중심: 데이터 분석, 인사이트 도출에 초점
- 협업 효율화 중심: 팀 간 소통, 정보 공유, 업무 흐름 개선에 초점

## 용어 기준
- 기술 용어 대신 비즈니스 용어 사용
  - "API 호출" → "외부 서비스 연동"
  - "시맨틱 검색" → "의미 기반 자료 검색"
  - "LLM" → "AI 언어 모델"
  - "RAG" → "자료 검색 기반 답변 생성"
  - "노드" → "처리 단계"
  - "프롬프트" → "AI 지시사항"
  - "파라미터" → "설정값"
  - "임베딩" → "문서 색인"
  - "토큰" → "처리 단위"
  - "폴백" → "대체 처리"
- 현업 담당자가 읽고 바로 이해할 수 있는 수준으로 작성
- 필요 시 괄호 안에 기술 용어를 병기 가능 (예: "의미 기반 자료 검색(시맨틱 검색)")

## 출력 형식
각 버전은 아래 8개 섹션을 포함하여 작성:

### 버전 N (관점: {관점명})

**1. 서비스 개요**
- 서비스명: {자동 생성}
- 서비스 유형: Workflow / Chatflow / Agent 중 가장 적합한 것 선택
- 서비스 목적: 해당 관점에서 재해석한 목적
- 대상 사용자: 구체적인 사용자 그룹

**2. 사용자 시나리오**
- 현재 상태(As-Is): 현재 업무 처리 방식과 문제점
- 목표 상태(To-Be): 에이전트 도입 후 업무 처리 방식
- 기대 효과: 정량적 수치 포함 (예: 처리 시간 50% 단축)

**3. 에이전트 역할 및 행동**
- 역할 정의: 에이전트가 수행하는 역할 한 줄 요약
- 단계별 행동: "사용자가 ~하면" 으로 시작하여 번호 매긴 단계별 기술
  각 단계마다 입력/처리/출력을 명시
- 예외 처리: 오류 상황별 대응 방법

**4. 워크플로우 설계**
- 입력 항목과 출력 항목 정의
- 분기 조건: "만약 ~이면 → ~처리" 형식
- 반복 처리: 해당하는 경우만 기술

**5. 외부 도구 및 데이터 소스**
- 필요한 도구/서비스 목록과 각각의 용도
- 자료 저장소: 사용할 문서/데이터 유형과 검색 방식
- 외부 연동: 연동할 서비스와 데이터 교환 방식

**6. AI 지시사항 가이드**
- AI에게 부여할 역할과 성격
- 응답 언어, 형식, 길이 기준
- 금지 사항이나 주의 사항

**7. 품질 요구사항**
- 응답 속도: 목표 시간
- 정확도: 목표 수치와 측정 기준
- 보안: 민감 정보 처리 방침
- 운영 관리: 모니터링, 알림 기준

**8. 검증 시나리오**
- 정상 케이스 2~3개 (입력 → 기대 결과)
- 예외 케이스 1~2개 (비정상 입력 → 기대 결과)

## 버전 간 차별화 규칙
- 서비스명은 관점을 반영하여 각각 다르게 명명
- 동일한 단계별 행동이라도 관점에 따라 우선순위와 상세도를 다르게 기술
- 품질 요구사항의 중점 지표를 관점별로 차별화
  - 자동화 중심 → 처리 속도와 처리량 강조
  - 고객 경험 중심 → 정확도와 만족도 강조
  - 비용 절감 중심 → 처리 건수 대비 비용 강조
- 검증 시나리오는 해당 관점의 핵심 가치를 검증하는 케이스로 구성
```
