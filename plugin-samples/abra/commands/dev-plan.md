# /abra:dev-plan

검증된 DSL과 시나리오를 기반으로 개발계획서를 작성함.

## 사용법

```
/abra:dev-plan
```

## 동작

이 명령은 `dev-plan` 스킬을 활성화하여 다음 워크플로우를 실행함:

1. **입력 확인**: 검증된 DSL 파일과 scenario.md 존재 확인
2. **비기능요구사항 수집**: 기술스택, 배포 환경, 성능, 보안 요구사항 질문
3. **개발계획서 작성**: plan-writer 에이전트에 위임하여 9개 섹션 포함 계획서 생성
4. **리뷰 포인트 체크**: DSL-계획 일관성, 비기능요구사항 반영, 전환 전략, 배포 실현 가능성 확인

## 산출물

- `{output_dir}/dev-plan.md` — 개발계획서 마크다운 파일

## 선행 조건

- 검증된 DSL 파일 존재 (STEP 3 완료)
- scenario.md 파일 존재 (STEP 1 완료)

## 다음 단계

개발계획서 검토 후 `/abra:develop` 호출하여 AI Agent 구현 시작.

---

**위임**: 이 명령은 `Skill` 도구를 통해 `abra:dev-plan` 스킬에 위임함.

- **INTENT**: 개발계획서 작성 (STEP 4) 수행
- **ARGS**: 사용자 요청 원문
- **RETURN**: 개발계획서 작성 완료 후 결과 보고
