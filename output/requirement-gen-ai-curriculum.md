## 핵심기능

## 사용자 플로우
- Step 1. 정보 수집: 사용자에게 문의하여 교육 관련 정보 수집
  - 교육 대상: PM/PO, 기획자, 개발자 등
  - 교육 형태: 일회성 특강 / 정식교육
    - 일회성 특강이면 시간 문의
    - 정식교육이면 며칠이고 하루 몇시간인지 문의
  - 차별성: 커리큘럼 생성에 적용할 차별성 요소
- Step 2. 교재 분석: 사용자가 제공한 경로의 교재를 분석
  - 교재 분석 체계를 추천하여 사용자 승인 요청
  - 승인된 체계에 따라 교재 분석 수행
- Step 3. 커리큘럼 생성
  - 커리큘럼 생성 수행 계획서 작성
  - 사용자에게 수행 계획서 검토 및 승인 요청
  - 커리큘럼 생성
  - 감성적인 완료 메시지로 사용자에게 알림
- Step 4. 프리젠테이션 작성
  - 프리젠테이션 대상 문의 (교육담당자, 수강생, 일반대중 등)
  - 프리젠테이션 수행 계획 작성
    - 대상자에 맞게 작성
    - 골든써클 접근법 사용: WHY → HOW → WHAT
    - 한 페이지에 핵심 메시지 3개 이하
    - 사용자에게 검토 및 승인 요청
  - 프리젠테이션 작성
    - 출력 형식: PPTX (PowerPoint)
    - 폰트: 맑은고딕
    - 텍스트 크기: 타이틀 22pt, 거버넌스 메시지 18pt, 본문 강조 16pt, 본문 일반 14pt 이상
    - 도식화와 이미지 활용 (필요 시 이미지 생성)

## 기술 요건

- 커리큘럼 출력 형식: Markdown (.md)
- 프리젠테이션 출력 형식: PPTX (python-pptx 커스텀 앱 개발 필요)
