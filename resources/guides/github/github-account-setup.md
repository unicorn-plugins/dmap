## GitHub 소개

GitHub는 Git 기반의 코드 호스팅 서비스임.
개인 및 팀 프로젝트의 소스 코드를 관리하고 공유할 수 있는 플랫폼을 제공함.

**주요 특징:**
- 무료 플랜으로 무제한 Public/Private 레포지토리 사용 가능
- 전 세계 개발자와 협업 및 오픈소스 기여 가능
- Issues, Pull Requests, Actions 등 다양한 협업 도구 제공
- 포트폴리오 및 개발 이력 관리에 활용 가능

## 계정 생성 절차

GitHub 계정 생성은 다음 단계로 진행됨:

1. **GitHub 웹사이트 접속**
   - 브라우저에서 [https://github.com](https://github.com) 접속

2. **회원가입 시작**
   - 우측 상단 `Sign up` 버튼 클릭

3. **기본 정보 입력**
   - **Email address**: 사용 중인 이메일 주소 입력
   - **Password**: 안전한 비밀번호 설정 (최소 15자 또는 8자 이상 + 숫자/특수문자)
   - **Username**: GitHub에서 사용할 고유한 사용자명 입력 (URL에 사용됨)
   - **Email preferences**: 제품 업데이트 수신 여부 선택 (y/n)

4. **계정 인증**
   - 사람인지 확인하는 퍼즐 또는 캡차 완료
   - `Create account` 버튼 클릭

5. **이메일 인증**
   - 입력한 이메일 주소로 발송된 인증 코드 확인
   - 인증 코드 입력하여 이메일 소유권 확인

6. **개인화 설정 (선택사항)**
   - 팀 규모, 학생/교사 여부, 관심 분야 등 입력
   - Skip 가능 (나중에 설정 가능)

7. **계정 생성 완료**
   - GitHub 대시보드 접근 가능

## 프로필 기본 설정

계정 생성 후 프로필을 설정하여 신뢰성과 가시성을 높일 수 있음:

1. **프로필 페이지 접속**
   - 우측 상단 프로필 아이콘 클릭 → `Your profile` 선택

2. **프로필 편집**
   - `Edit profile` 버튼 클릭

3. **기본 정보 입력**
   - **Name**: 실명 또는 닉네임 입력
   - **Bio**: 간단한 자기소개 (최대 160자)
   - **Company**: 소속 회사 또는 조직
   - **Location**: 거주 지역
   - **Website**: 개인 블로그 또는 포트폴리오 사이트 URL
   - **Twitter**: 트위터 계정 연동 (선택사항)

4. **프로필 이미지 설정**
   - 좌측 상단 아바타 클릭 → 이미지 업로드

5. **저장**
   - `Save` 버튼 클릭하여 변경사항 저장

## 주의사항

GitHub 계정 사용 시 다음 사항에 유의함:

**사용자명 관련:**
- 사용자명은 GitHub URL에 사용됨 (예: `github.com/{username}`)
- 변경 가능하나 기존 URL은 자동 리다이렉트되지 않음
- 다른 사용자가 기존 사용자명을 선점할 수 있음
- 신중하게 선택하거나 초기에 확정하는 것을 권장

**보안 관련:**
- **2FA(Two-Factor Authentication) 활성화 권장**
  - Settings → Password and authentication → Two-factor authentication
  - 인증 앱(Google Authenticator, Authy 등) 또는 SMS 사용
  - 계정 탈취 위험을 크게 낮춤
- 비밀번호는 정기적으로 변경
- 비밀번호를 다른 서비스와 공유하지 않음

**이메일 주소 관련:**
- 기본 이메일 주소는 커밋 이력에 표시됨
- Private email 기능 활성화 가능:
  Settings → Emails → Keep my email addresses private 체크
- 여러 이메일 주소를 등록하여 커밋별로 다른 이메일 사용 가능

**계정 플랜:**
- Free 플랜으로도 대부분의 기능 사용 가능
- Private 레포지토리 무제한 생성 가능
- 협업자 수 제한 없음
- 필요 시 Pro/Team/Enterprise 플랜으로 업그레이드 가능
